# Credit Risk Platform

A data engineering practice project simulating a banking credit risk evaluation platform.
Built with dbt, Apache Airflow, PostgreSQL, and Docker.

## Tech Stack

- **Python 3.13**
- **PostgreSQL 15**
- **dbt**
- **Apache Airflow 3.1.8** 
- **Docker & Docker Compose**

## Setup Steps

### 1. Clone the repository
```bash
git clone <repo-url>
cd credit_risk_platform
```

### 2. Create environment file
```bash
cp .env.example .env
# Fill in your credentials in .env
```

### 3. Start Docker containers
```bash
docker-compose up --build -d
```

### 4. Check all containers are running
```bash
docker-compose ps
```

### 5. Access Airflow UI
- URL: http://localhost:8080
- credentials are auto-generated by Airflow 3.x SimpleAuthManager
- Find the password in the logs:
```bash
docker-compose logs airflow-api-server | grep password
```

## Models
### Staging Layer
Cleans and standardizes raw data, materialized as views:
- `stg_customers` вЂ” cleaned customer profiles
- `stg_loan_products` вЂ” cleaned loan product catalog
- `stg_credit_score_bands` вЂ” credit score bands and risk levels
- `stg_loan_applications` вЂ” cleaned loan applications from mock API
- `stg_transactions` вЂ” cleaned transactions from mock API

### Intermediate Layer
Business logic combining staging models, materialized as views:
- `int_customer_risk_profile` вЂ” joins customers with credit score bands and loan history
- `int_loan_application_summary` вЂ” enriches loan applications with product details
- `int_customer_transactions_summary` вЂ” aggregates transaction history per customer

### Mart Layer
Final analytics-ready tables, materialized as tables:
- `mart_credit_risk_assessment` вЂ” full risk picture per customer including debt category and risk score
- `mart_loan_portfolio` вЂ” loan portfolio summary grouped by product type and risk category
- `mart_customer_360` вЂ” complete customer view combining risk, transactions and latest application

### dbt Tests
48 tests covering:
- Unique and not null constraints
- Accepted values for categorical columns
- Data quality across all three layers

### Loading Mock API Data
Before running dbt models, load the mock API data into PostgreSQL:
```bash
docker-compose run python python /opt/scripts/load_api_data_to_postgres.py
```

### Running dbt

#### Load seed data
```bash
docker-compose run dbt dbt seed --project-dir /opt/dbt/credit_risk --profiles-dir /opt/dbt
```

#### Run all models
```bash
docker-compose run dbt dbt run --project-dir /opt/dbt/credit_risk --profiles-dir /opt/dbt
```

#### Run tests
```bash
docker-compose run dbt dbt test --project-dir /opt/dbt/credit_risk --profiles-dir /opt/dbt
```

#### Run specific layer
```bash
docker-compose run dbt dbt run --select staging --project-dir /opt/dbt/credit_risk --profiles-dir /opt/dbt
```
